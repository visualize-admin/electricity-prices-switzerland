# i18n

LinguiJS with `.po` files per locale (`de`, `fr`, `it`, `en`). Translations managed via [Accent](https://accent.interactivethings.io).

## Workflow for new keys

**1. Extract** — adds new empty entries, removes obsolete ones:
```bash
pnpm locales:extract
```

**2. Machine-translate** empty strings with DeepL:
```bash
tpo translate
# --dry-run to preview, --only <lang> to restrict, --formality more for formal register
```

**3. Check for inconsistencies with Claude** — paste this prompt with the `.po` diff:

```
Swiss electricity price website. New msgstr values added in the diff below.
Flag: (1) terminology drift vs existing keys, (2) translation errors (duplicate words,
wrong word sense, incomplete phrases), (3) within-diff inconsistency (e.g. "rabais"
vs "réduction" for the same concept). For each issue: msgid, current msgstr, fix,
reference key.
```

See [terminology.md](./terminology.md) for canonical terms. Key reference sections in the `.po` files:
- `selector.pricecomponent.*` — canonical short labels
- `sunshine.export.column.*` — established export column headers


**4. Compile:**
```bash
pnpm locales:compile
```
